## 0. Decision

Use **Demo-first SaaS Skeleton**.

The first build must prove the product loop:

> Product image + selling points -> 3 English ad concepts -> 1 vertical product ad video -> downloadable result.

It should feel like a real SaaS product, but it should not spend the first 5 days building a full billing system.

What is in scope:

- Next.js app.
- Generator-first landing page.
- Creative package generation with structured JSON output.
- One video generation provider behind a provider adapter.
- Supabase database for generations and creative packages.
- Cloudflare R2-ready storage abstraction.
- Basic auth-ready architecture.
- Analytics events.
- Five SEO pages.

What is deferred:

- Real subscriptions and payment webhooks.
- Full credits economy.
- Multi-provider routing.
- Team accounts.
- User-facing API.
- Self-owned model training or fine-tuning.

## 1. Architecture

```text

  | upload product image + product fields
  v

  |
  | /api/creative-packages
  v

  |
  | LLM structured JSON
  v

  |
  | stores product input, 3 ad angles, selected concept
  v
Next.js Result UI
  |
  | /api/generations
  v

  |
  | provider job id / polling
  v

  |
  | mp4 url
  v

  |
  | MVP: remote provider URL or R2 object
  v

```

For MVP, the app can poll generation status from the browser. Queue + webhook is the upgrade path after real usage exists.

## 2. Technical Stack

| Layer | First choice | Why | MVP cost | 1000 DAU cost | Replace when |
|---|---|---|---:|---:|---|
| Frontend | Next.js 15 App Router | One app for generator, result, SEO pages, and API routes | $0 | $0-20 on Vercel | API routes hit timeout or traffic limits |
| UI | Tailwind CSS + shadcn/ui + Radix | Fast build, good forms/dialogs/tabs, easy visual polish | $0 | $0 | Custom design system is justified by brand needs |
| API | Next.js Route Handlers | Least infrastructure for 5-day MVP | $0 | $0-20 | Long-running jobs require queue workers |
| DB | Supabase Postgres | Fast setup, SQL, future auth/RLS support | $0 | $25+ | DB > 500MB or sustained production traffic |
| Auth | Supabase Auth, initially optional | Same vendor as DB, Google OAuth later | $0 | $0-25 | Need enterprise auth or advanced user management |
| Storage | Cloudflare R2 | Cheap video/object storage, no egress tax | $0 | $1-20 excluding API usage | Need media transforms/CDN features |
| Video API | fal.ai or similar aggregator | Fastest path to image-to-video provider testing | Usage-based | Usage-based | Output quality/cost forces direct model API |
| LLM | OpenAI-compatible structured output | Reliable JSON for creative packages | Usage-based | Usage-based | Prompt cost or quality requires smaller/local model |
| Analytics | PostHog or Vercel Analytics | Funnel visibility for validation | $0 | $0-20 | Need warehouse-grade analytics |
| Errors | Sentry | Catch failed generations and API errors | $0 | $0-26 | Team/volume grows |
| Email | Resend | Email capture and later transactional emails | $0 | $0-20 | High email volume |
| Payments | Deferred: Creem/DodoPayment | Avoid blocking 5-day validation | $0 | Fees only | Users ask to pay or usage costs need gating |

## 3. Page and Route Map

### Public Pages

- `/` - generator-first landing page.
- `/generate` - optional dedicated generator route if home becomes marketing-heavy later.
- `/results/[id]` - result page with creative package, video preview, download, retry.
- `/ai-product-video-generator`
- `/shopify-product-video-generator`
- `/tiktok-product-video-generator`
- `/ai-video-ad-generator`
- `/product-image-to-video`
- `/privacy`
- `/terms`
- `/contact`

### API Routes

- `POST /api/uploads/product-image`
  - Accept product image.
  - Return temporary URL or stored asset id.

- `POST /api/creative-packages`
  - Input product fields and image reference.
  - Call Creative Engine.
  - Persist package and concepts.
  - Return package id and three ad concepts.

- `POST /api/generations`
  - Input package id and selected concept id.
  - Create generation row.
  - Call video provider adapter.
  - Return generation id and provider job id.

- `GET /api/generations/[id]`
  - Return generation status, progress if available, output URL when complete.

- `POST /api/generations/[id]/retry`
  - Retry failed generation from the same selected concept.

- `POST /api/leads`
  - Capture email and context after result/download.

## 5. Creative Engine

The Creative Engine is the product's main differentiator. It should return strict JSON, not free-form text.

Input:

```json
{
  "productName": "Portable Neck Fan",
  "category": "consumer gadgets",
  "sellingPoints": [
    "hands-free cooling",
    "quiet motor",
    "10-hour battery"
  ],
  "targetAudience": "commuters and outdoor workers",
  "offer": "20% off this week",
  "platform": "tiktok_reels_meta",
  "tone": "direct_response"
}
```

Output:

```json
{
  "concepts": [
    {
      "angle": "pain",
      "hook": "Still sweating through your commute?",
      "storyboard": [
        {"scene": 1, "visual": "Close-up of a hot commuter outdoors", "text": "Hot commute?"},
        {"scene": 2, "visual": "Product around neck with airflow motion", "text": "Hands-free cooling"},
        {"scene": 3, "visual": "Product and discount offer", "text": "20% off this week"}
      ],
      "caption": "Stay cool anywhere with a quiet wearable fan built for long days.",
      "cta": "Shop the cooling fan today",
      "thumbnailPrompt": "...",
      "videoPrompt": "..."
    }
  ]
}
```

Validation rules:

- Exactly 3 concepts.
- Angles must be `pain`, `benefit`, `urgency`.
- Each concept must have 3 scenes.
- Hooks should be <= 12 words.
- On-screen text should be short enough for mobile.
- Output must be English.

If JSON validation fails, retry once with a repair prompt. If repair fails, return a clear error and do not call the video provider.

## 6. Video Generation Pipeline

### MVP Flow: Polling

Use polling first.

1. User selects a concept.
2. Frontend calls `POST /api/generations`.
3. Server creates a `generations` row with `queued`.
4. Server calls the provider adapter.
5. Provider returns a job id or immediate result.
6. Frontend polls `GET /api/generations/[id]` every 2 seconds.
7. API checks provider status if needed and updates DB.
8. Result page shows video when status is `succeeded`.

This is enough for DAU < 500 and peak concurrent generations < 20.

### Upgrade Flow: Queue + Webhook

When there is real traffic:

- Add Inngest, Trigger.dev, or Upstash QStash.
- Move generation calls out of request lifecycle.
- Add webhook endpoint per provider.
- Store webhook ids in `webhooks_log`.
- Add automatic retry and fallback provider.

## 7. Provider Adapter

Do not let product code call a provider directly.

```ts
export type GenerationInput = {
  productImageUrl: string;
  videoPrompt: string;
  aspectRatio: "9:16";
  durationSeconds: 10 | 15;
  seed?: number;
};

export type GenerationJob = {
  provider: string;
  providerJobId: string;
  status: "queued" | "running" | "succeeded" | "failed";
  outputUrl?: string;
  costUsd?: number;
  errorMessage?: string;
};

export interface VideoProvider {
  name: string;
  create(input: GenerationInput): Promise<GenerationJob>;
  getStatus(providerJobId: string): Promise<GenerationJob>;
}
```

Initial implementation:

- `FalVideoProvider` or another API provider chosen during implementation.

Future implementations:

- `ReplicateVideoProvider`
- `KlingDirectProvider`
- `SeedanceDirectProvider`
- `InternalModelProvider`

## 8. Storage Strategy

MVP:

- Product images can be uploaded to Supabase Storage or R2.
- Generated MP4 can initially use provider-hosted URL if the provider returns a stable URL.
- If provider URLs expire, copy completed MP4 to R2 immediately.

Preferred architecture:

- R2 bucket: `product-ad-media`.
- Object keys:
  - `product-images/{session_or_user}/{asset_id}.jpg`
  - `generated-videos/{session_or_user}/{generation_id}.mp4`
  - `thumbnails/{session_or_user}/{asset_id}.jpg`

Access:

- Keep bucket private.
- Serve downloads via signed URLs.
- Free/anonymous outputs expire after 7 days.
- Authenticated or paid outputs can be retained longer.

## 9. Auth and Credits

For the 5-day MVP:

- Allow anonymous first generation if API cost is controlled.
- Capture email after result or before download.
- Store `session_id` on all rows.
- Add `user_id` columns now so auth can be added without migration pain.

Do not implement real paid credits in the first sprint.

Prepare for credits:

- `generations.cost_usd` must be recorded.
- Provider adapter must return or estimate cost.
- Add `credits_transactions` when payment is introduced.

Future credit behavior:

- Deduct credits when generation starts.
- Refund automatically if generation fails.
- Store all balance changes in `credits_transactions`.
- Do not trust frontend for credit balance or price.

## 10. Security and Abuse Controls

Minimum controls:

- File type allowlist: jpg, png, webp.
- File size limit: 10 MB.
- Rate limit generation routes by IP/session.
- Max one active generation per anonymous session.
- Server-side API keys only. Never expose provider keys to browser.
- Basic prompt safety: reject illegal, adult, or clearly abusive ad requests.
- Log provider errors without logging secrets.

User-visible failure rules:

- Creative package failure: show a clear retry action.
- Video failure: preserve the creative package and allow retry.
- Provider timeout: mark failed, tell user no charge/credit loss.

## 15. Capacity Plan

| Stage | DAU | Daily generations | Peak concurrent jobs | Main bottleneck | Upgrade |
|---|---:|---:|---:|---|---|
| MVP | < 50 | < 100 | < 5 | Provider reliability | Manual monitoring |
| Early | 50-500 | 100-1000 | 5-20 | API route polling and provider cost | Add queue/webhook |
| Growth | 500-5000 | 1000-10000 | 20-100 | Job orchestration and storage lifecycle | Inngest/Trigger + R2 lifecycle |
| Scale | 5000+ | 10000+ | 100+ | Cost, abuse, provider quota | Multi-provider routing and paid credits |

Thresholds:

- Add queue when peak concurrent jobs > 20.
- Add paid credits before daily provider cost exceeds comfort budget.
- Add provider fallback when failed generations exceed 5%.
- Add auth requirement if abuse appears or anonymous cost grows.

## 16. Main Risks

### Risk 1: Video provider quality is not good enough

Mitigation:

- Preserve the ad package value even if video is weak.
- Let user retry from the same concept.
- Test 3-5 ecommerce categories before launch.

### Risk 2: API cost grows before payment exists

Mitigation:

- Rate limit anonymous generation.
- Consider one free generation per session.
- Capture email before expensive retry.
- Add paid credits as soon as users ask for repeated use.

### Risk 3: Product becomes generic

Mitigation:

- Make Creative Engine the center of the UX.
- Show three ad angles before video generation.
- Add category-specific templates quickly.

### Risk 4: 5-day scope creep

Mitigation:

- No subscription system in first sprint.
- No multi-provider routing in first sprint.
- No domestic platform templates in first sprint.
- No batch generation in first sprint.

### Risk 5: No real demand despite working product

Mitigation:

- Launch with manual outreach to 20 sellers.
- Ask users for real product images.
- Count usage and repeated requests, not compliments.

## 18. Approval Criteria

This technical architecture is approved when we agree:

- First sprint prioritizes product loop over billing.
- Provider adapter is mandatory even with only one provider.
- Supabase stores all core generation state.
- R2 is the intended storage path for generated videos.
- Payments and full credits are deferred until validation.
